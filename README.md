# Duty Management System (Vengg)

ระบบบริหารจัดการ การจัดเวรและรายงานเวร (Vengg) พัฒนาขึ้นเพื่อช่วยอำนวยความสะดวกในการจัดตารางเวร การเปลี่ยนเวร และการรายงานผลการปฏิบัติหน้าที่ของหน่วยงาน

## 🌟 คุณสมบัติเด่น (Features)
- **ระบบปฏิทินเวร (Calendar View)**: แสดงตารางเวรในรูปแบบปฏิทินที่ดูง่าย พร้อมระบุสีแยกประเภทเวรชัดเจน
- **ระบบรายการเวร (List View)**: แสดงผลในรูปแบบรายการที่กรองตามวัน เดือน ปี และประเภทเวรได้แม่นยำ
- **ระบบสถิติการอยู่เวร**: แสดงจำนวนเวรแยกตามประเภทและกลุ่มหน้าที่ในแต่ละเดือน
- **ระบบจัดการการเปลี่ยนเวร (Duty Swapping)**: รองรับการขอเปลี่ยนเวรและการโอนเวร (ยกเวรให้) พร้อมระบบอนุมัติจากฝ่ายอำนวยการ
- **ระบบรายงาน (Reporting)**: สร้างรายงานผลการปฏิบัติหน้าที่ (เช่น รายงานเวรหมายจับหมายค้น) ในรูปแบบไฟล์เอกสาร
- **ระบบบริหารจัดการ (Admin Module)**:
    - จัดเวรอย่างรวดเร็ว (Quick Set)
    - จัดการวันหยุดราชการ
    - จัดการสมาชิกและกำหนดสิทธิ์ (RBAC)
    - ตั้งค่าการแจ้งเตือนผ่าน Line Notify

## 🛠 เทคโนโลยีที่ใช้ (Tech Stack)
- **Frontend**: HTML5, CSS3, Bootstrap 5, Vue.js 3, FullCalendar v5, Axios
- **Backend**: PHP (รองรับ 7.4 ขึ้นไป), PDO สำหรับเชื่อมต่อฐานข้อมูล
- **Database**: MySQL / MariaDB
- **Libraries**: SweetAlert2 (แจ้งเตือน), Perfect Scrollbar

## 📂 โครงสร้างโฟลเดอร์ (Project Structure)
- `/assets`: เก็บไฟล์ CSS, JavaScript, รูปภาพ และฟอนต์ที่ใช้ในระบบ
- `/database`: ไฟล์ SQL สำหรับนำเข้าฐานข้อมูล
- `/pages`: ไฟล์ UI หลักของระบบ แบ่งเป็นโมดูลต่างๆ (dashboard, asu, users, history, fnu)
- `/server`: ไฟล์ Logic หลังบ้าน, การจัดการ API และการเชื่อมต่อฐานข้อมูล
- `/uploads`: เก็บไฟล์เอกสารที่ระบบสร้างขึ้นและรูปภาพโปรไฟล์
- `/vendor`: ไฟล์ Library ของ PHP

## 🖥 สเปกคอมพิวเตอร์ที่แนะนำ (System Requirements)

### 1. ฝั่งเซิร์ฟเวอร์ (Server Side)
- **CPU**: Dual Core 2.0 GHz หรือสูงกว่า
- **RAM**: 4 GB หรือสูงกว่า
- **Disk Space**: อย่างน้อย 1 GB (ขึ้นอยู่กับจำนวนรูปภาพและเอกสารที่อัปโหลด)
- **OS**: Windows (รองรับผ่าน XAMPP/WampServer) หรือ Linux (Ubuntu/CentOS)

### 2. ซอฟต์แวร์ที่ต้องติดตั้ง (Software Requirements)
- **Web Server**: Apache หรือ Nginx
- **PHP**: เวอร์ชัน 7.4 หรือ 8.x
- **Database**: MySQL 5.7+ หรือ MariaDB 10.4+
- **Browser**: Google Chrome, Microsoft Edge, หรือ Safari เวอร์ชันล่าสุด

## 🚀 วิธีการติดตั้ง (Installation)
1. ติดตั้งจำลองเซิร์ฟเวอร์ (เช่น XAMPP)
2. คัดลอกโฟลเดอร์โครงการไปไว้ใน `htdocs`
3. สร้างฐานข้อมูลใหม่ใน phpMyAdmin
4. นำเข้าไฟล์ SQL จากโฟลเดอร์ `/database`
5. ตั้งค่าการเชื่อมต่อฐานข้อมูลในไฟล์ `/server/connect.php`
6. เข้าใช้งานระบบผ่านเบราว์เซอร์ที่ `http://localhost/vengg`
